I -Step-By-Step Example: Data Scraping with OCRExtract

 

 

Data Scraping is the removal of useful data from a human-readable source. In our case, removing text from this image of a limerick.

 

To do this, we will select the command OCRExtract and pass in the image as an input.

 

OCRExtract will remove the text from an image input (referenced in the Target field), and attempt to store it in the Value field as a variable.

 

If you are not familiar with variables, see below.

 

 

To pass an image as an input:

1.    In the Visual tab, click Load Image

 

 

2.    Locate the image via File Explorer

3.    I would recommend you rename the file by clicking the pencil icon by the name

 

 

4.    Type the file name into the Target field of the OCRExtract command

 

 

5.    Type into the Value field a name we will use to reference the data we just harvested from the image.

 

 

 

To store the output in a variable to be returned to the user:

1.    Make a new echo command

 

 

2.    Put the name of the variable in the Target field, in the format of ${name}, which specifies to the computer that it is a variable name, and not just text

 

 

Note: Make sure the variable syntax ${} uses curly brackets, not curved ones.

 

On Variables

To access any variable value in the Target field (whether in echo or XClickText or anything), the following syntax should be used:

 

${YourVariable}  

 

Make sure you use curly brackets {} and not curved brackets ()!

 

Some variables are built into UI.Vision, such as:

${KEY_EXC}                  Escape Key

${KEY_WIN}                  Windows Key

[All of these can be seen on the documentation for XType under Info for this Command, as explained in I.C.2]

 

Thus, you can not name one of your variables the same name as these preset values.

The name is already taken!

 

II. Step-By-Step Example: Windows WordPad

1. Open WordPad

To run WordPad, we will use the XRun command to open Wordpad.

 

XRun opens an executable file on your computer, functionally identical to opening file explorer and double-clicking on the .exe file.

To do this, we need the full File Path of the .exe file.

(Reminder: The full file path is the path from the starting directory to the folder containing the file, combined with the file name and its file extension.)

 

E.g:

●      C:\ is the starting directory

●      Program Files\Windows NT\Accessories\ is the path to the folder containing WordPad

●      wordpad is the name of the executable

●      .exe. Is the file extension\

 

Combining these, we get the full File Path of WordPad:

3. Get your output

To obtain our output, we will be using OCRExtractRelative.

Relative commands function above and beyond its non-relative version. They are more precise and reliable, but can only be used when the surroundings to the selected area are always the same! (E.g a programs UI and not your desktop!)

 

Relative commands which use images must have additional setup steps:

Open the Visual tab under the space to code your macro.

 

On Relative Clicks

 

Click the eye icon to open the relative clicks menu. In this menu, you will split the image into two non-overlapping areas.:

●      The anchor area

●      The click area

 

The click area is where the click will occur.

The anchor area is the unchanging area of the UI.

 

You can only make one of each area, so just cover as much applicable area as possible for the anchor image area.

On the other hand, the click area should be smaller and more precise.
